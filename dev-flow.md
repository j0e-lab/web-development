# 開発するうえで実施されるフロー

## 1. 要件定義

まず「何を作るのか」を決める。

- 機能一覧
- ユーザー種別
- 画面数
- API仕様
- 非機能要件（性能・セキュリティなど）

ここを飛ばすと後の設計が破綻しやすい。

## 2. 画面設計（UI設計）

フロントエンドの見た目や画面構成を設計する。

- ワイヤーフレーム
- 画面遷移図
- UIレイアウト

## 3. データ設計（DB設計）

データベースの構造を決める。

- テーブル設計
- カラム設計
- インデックス設計
- 正規化
- ER図作成

## 4. API / バックエンド設計

サーバー側の処理やAPI仕様を設計する。

- APIエンドポイント
- HTTPメソッド
- リクエスト/レスポンス形式
- 認証方式
- エラーハンドリング

## 5. 実装

ここで初めてコードを書く。

一般的な実装順序

1. DB作成
2. Backend実装
3. Frontend実装

## 6. テスト

実装後に動作確認を行う。

- 単体テスト
- APIテスト
- UIテスト
